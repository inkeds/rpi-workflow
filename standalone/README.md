# 单文件快速启动

如果暂时不想安装完整 RPI Workflow，可以只下载一个 Markdown 放入项目根目录，让 Agent 完成材料审计、关键决策、V1 边界、设计初始化、AGENTS 规则和后续功能变更协议。

[查看或下载《AI项目初始化与持续迭代协议.md》](AI项目初始化与持续迭代协议.md)

## 适合场景

- 新项目只有想法、需求文档或参考材料，需要先形成可信设计。
- 已有项目缺少稳定的 AGENTS.md、设计入口和功能变更规则。
- 希望先让 AI 审计和设计，不立即安装完整工作流。
- 希望下载一个文件后即可在 Codex、Claude Code 或通用 Agent 中启动。

## 初始化项目

下载协议文件并放入目标项目根目录，然后对 Agent 说：

```text
请完整阅读《AI项目初始化与持续迭代协议.md》，执行“项目初始化协议”。
需求和参考材料位于：docs/requirements/
请先完成审计和设计；遇到 P0 决策时先询问我，不要直接写代码。
```

Agent 会先盘点材料、识别冲突和隐藏产品模式，再确认关键决策、定义完整 V1，并生成适合当前项目的仓库规则与设计入口。

## 只阅读和设计

如果只需要分析方案，不希望修改仓库：

```text
请完整阅读《AI项目初始化与持续迭代协议.md》，使用 audit-only 模式。
阅读 README.md、prd.md 和现有代码，分析真实诉求、冲突、V1 边界和推荐设计，不要修改文件。
```

## 初始化后增加功能

初始化完成后，项目应由自身 AGENTS.md 和设计文档接管，不需要每次重新读取整份协议：

```text
请按照仓库 AGENTS.md 中的“功能变更协议”处理：
为当前产品增加批量导出功能，先完成影响分析和 Spec，再实现并验证。
```

## 与完整 RPI Workflow 的区别

单文件版与 RPI 共享以下原则：

- 区分用户事实、AI 推断、假设和正式决策。
- 新功能先做影响分析，不静默覆盖基线和不变量。
- 设计描述预期，代码和测试描述当前实现。
- 功能完成前对账设计、实现、测试和迁移。

单文件版是一份执行协议，不模拟完整 Workflow 的文件锁、事务恢复、Schema 校验、Hook、CLI、增量索引和 Eval。出现以下情况时，建议升级到完整 RPI Workflow：

- 多个 Agent 长期并发开发。
- 需要自动门禁、任务状态和可恢复事务。
- 需要持续 Eval、Schema 校验和变更追踪。
- 项目规模较大，设计漂移和跨域修改风险明显增加。

## 初始化完成后

协议文件可以继续保留作为参考，也可以由用户移除。项目后续开发应依赖自身 AGENTS.md、设计文档、代码、迁移和测试，不依赖原始聊天或协议文件持续存在。
