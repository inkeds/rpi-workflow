# M2 — 工程护栏与上线准备

**Vibe:Spec = 2:8**（规范主导，方向已定）

## 比例含义

| 维度 | 2（Vibe） | 8（Spec） |
|------|-----------|-----------|
| 重心 | 上线策略微调 | 可观测、可回滚、安全审计 |
| 容忍 | 小范围 UX 优化 | 不允许功能变更 |
| 禁止 | 新增功能、重构核心 | 跳过安全扫描 |

## M2 入场检查清单

- [ ] M1 所有 Task 已 closed（result=pass）
- [ ] 关键路径有日志/指标埋点
- [ ] 回滚方案已文档化（spec 中有 rollback 章节）
- [ ] 安全扫描已运行（无 high/critical 漏洞）
- [ ] e2e 测试骨架已建立

## M2 出场验收标准

- [ ] e2e 测试通过（核心用户路径）
- [ ] 安全扫描通过（`npm audit` / `pip-audit` 等）
- [ ] 可观测性校验：关键指标可查询
- [ ] 部署脚本/CI 流程已验证
- [ ] 故障恢复流程已演练（至少文档层面）
- [ ] 质量门控全部通过（所有 M2 gates）

## 典型 M2 反模式（需立即停止）

- 在 M2 阶段添加"顺手做的"新功能
- 跳过 e2e 直接发布
- 安全扫描报 high 漏洞却标记为已接受
- 没有回滚方案就部署到生产
