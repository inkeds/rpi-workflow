# M2 Operate — 长期运营与治理

> 根据真实规模和风险建立安全、可观测、审计、容量和成本治理。

## 阶段姿态

- 产品：依据真实运营数据决定继续优化、扩展或保持范围。
- 工程：建立可观测、可回滚、安全审计、容量和成本治理。
- 范围：允许有证据的新能力，但必须独立进入新的探索/验证循环。
- 运营：不同项目按实际风险选择治理项，不强迫个人本地工具模拟企业架构。

## M2 入场检查清单

- [ ] M1 所有 Task 已 closed（result=pass）
- [ ] 关键路径有日志/指标埋点
- [ ] 回滚方案已文档化（spec 中有 rollback 章节）
- [ ] 安全扫描已运行（无 high/critical 漏洞）
- [ ] e2e 测试骨架已建立

## M2 出场验收标准

- [ ] e2e 测试通过（核心用户路径）
- [ ] 安全扫描通过（`npm audit` / `pip-audit` 等）
- [ ] 可观测性校验：关键指标可查询
- [ ] 部署脚本/CI 流程已验证
- [ ] 故障恢复流程已演练（至少文档层面）
- [ ] 质量门控全部通过（所有 M2 gates）

## 典型 M2 反模式（需立即停止）

- 在 M2 阶段添加"顺手做的"新功能
- 跳过 e2e 直接发布
- 安全扫描报 high 漏洞却标记为已接受
- 没有回滚方案就部署到生产
