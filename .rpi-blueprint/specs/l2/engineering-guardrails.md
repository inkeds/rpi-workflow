# L2 工程护栏

1. 输出结构约束：Schema 校验 + 错误处理一致性
2. 产物追溯约束：任务 ID、spec_refs、context_refs 必须可回放
3. 安全约束：密钥管理、最小权限、敏感信息脱敏
4. 运行约束：限流、重试、幂等、故障恢复
5. 质量约束：verify + gates 双层门控
6. Monorepo 约束：优先 affected 范围执行测试与检查
7. 变更约束：功能修改必须关联 CHG/DEC；未解决的高影响决策失败闭锁
8. 设计约束：Capability、Invariant、Spec 与实现必须在任务关闭前对账
9. 状态约束：治理 JSON 必须通过 Schema 校验；迁移幂等且不得覆盖未来版本状态
10. 并发约束：共享状态使用项目级锁、原子写入和可恢复事务，禁止读改写竞态
11. 索引约束：项目扫描可增量复用，但缓存损坏必须自动重建且不能改变事实判定
12. 平台边界：`.rpi/` 为平台无关定义；适配目录不得保存共享事实
