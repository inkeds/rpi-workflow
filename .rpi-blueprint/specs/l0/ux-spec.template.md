## 二、表格 CRUD 标准实现

### 强制规则

1. **表格增删改查必须使用弹窗/抽屉**，禁止使用同级表单块
2. **操作按钮位置**：
   - 新增按钮：表格右上角
   - 编辑/删除按钮：表格每行的操作列
3. **弹窗/抽屉选择标准**：
   - 表单字段 ≤ 8 个：使用弹窗（Modal/Dialog）
   - 表单字段 > 8 个：使用抽屉（Drawer）
4. **删除操作必须二次确认**

### 标准实现模式

```
┌─────────────────────────────────────────┐
│  [新增] [批量删除] [导出]        [搜索框] │
├─────────────────────────────────────────┤
│  表格内容                                │
│  ┌──────┬──────┬──────┬────────┐       │
│  │ 姓名 │ 年龄 │ 状态 │ 操作   │       │
│  ├──────┼──────┼──────┼────────┤       │
│  │ 张三 │ 25   │ 启用 │ [编辑] │       │
│  │      │      │      │ [删除] │       │
│  └──────┴──────┴──────┴────────┘       │
│                                         │
│  [分页组件]                              │
└─────────────────────────────────────────┘

点击 [新增] 或 [编辑] 后：
┌─────────────────────────────────────────┐
│  新增用户                          [×]   │
├─────────────────────────────────────────┤
│  姓名：[__________]                     │
│  年龄：[__________]                     │
│  状态：[下拉选择]                        │
│                                         │
│              [取消] [确定]               │
└─────────────────────────────────────────┘
```

### 禁止的实现方式

❌ **禁止**：在表格下方或旁边放置同级表单块
```
表格
[表单输入框] [提交按钮]  ← 禁止这种方式
```

## 三、表单设计规范

### 布局规则

1. **表单布局**：
   - 简单表单（≤ 5 个字段）：单列布局
   - 复杂表单（> 5 个字段）：双列布局
   - 超长表单：分步骤表单（Steps）

2. **标签对齐**：
   - 默认：右对齐
   - 移动端：顶部对齐

3. **必填标识**：使用红色星号 `*` 标记

### 校验规则

1. **实时校验**：失焦时校验
2. **提交校验**：提交前全量校验
3. **错误提示**：字段下方显示红色错误文本

### 反馈机制

1. **提交成功**：
   - 显示成功提示（Toast/Message）
   - 关闭弹窗/抽屉
   - 刷新列表数据

2. **提交失败**：
   - 显示错误提示
   - 保持弹窗/抽屉打开
   - 高亮错误字段

## 四、弹窗/抽屉使用规则

### 弹窗（Modal/Dialog）

**适用场景**：
- 表单字段 ≤ 8 个
- 确认操作（删除、提交等）
- 简单信息展示

**规范**：
- 宽度：600px（小）/ 800px（中）/ 1000px（大）
- 必须有标题
- 必须有关闭按钮（右上角 ×）
- 底部按钮：[取消] [确定]，确定按钮在右侧

### 抽屉（Drawer）

**适用场景**：
- 表单字段 > 8 个
- 详情页展示
- 多步骤表单

**规范**：
- 宽度：50%（中）/ 70%（大）
- 从右侧滑出
- 必须有标题和关闭按钮
- 底部固定操作栏

## 五、按钮布局规范

### 主要操作按钮

1. **位置**：
   - 页面级操作：页面右上角
   - 表格操作：表格右上角
   - 表单操作：表单底部右侧

2. **样式**：
   - 主要操作：Primary 按钮（蓝色）
   - 次要操作：Default 按钮（灰色）
   - 危险操作：Danger 按钮（红色）

3. **顺序**：
   - 左侧：次要操作（取消、返回）
   - 右侧：主要操作（确定、提交）

### 操作列按钮

1. **数量限制**：最多 3 个直接显示，超过使用下拉菜单
2. **顺序**：查看 > 编辑 > 删除
3. **样式**：文字按钮（Text Button）

## 六、数据展示规范

### 列表/表格

1. **分页**：
   - 默认每页 10 条
   - 支持切换：10 / 20 / 50 / 100
   - 分页器位置：表格底部右侧

2. **排序**：
   - 支持单列排序
   - 排序图标：表头右侧

3. **筛选**：
   - 简单筛选：表格右上角搜索框
   - 高级筛选：展开的筛选表单

### 状态展示

1. **状态标签**：使用 Tag/Badge 组件
   - 成功/启用：绿色
   - 警告/待审核：橙色
   - 错误/禁用：红色
   - 默认/草稿：灰色

2. **空状态**：
   - 显示空状态图标和文案
   - 提供引导操作（如"立即创建"）

## 七、反馈体系

### 操作反馈

1. **成功提示**：
   - 类型：Toast/Message（3 秒自动关闭）
   - 文案：简洁明确（如"保存成功"）

2. **错误提示**：
   - 类型：Toast/Message（需手动关闭或 5 秒后关闭）
   - 文案：说明错误原因和解决方案

3. **加载状态**：
   - 按钮加载：按钮显示 Loading 图标
   - 页面加载：显示 Skeleton 或 Spin 组件

### 确认操作

1. **删除确认**：
   - 使用 Popconfirm 或 Modal
   - 文案：明确说明删除后果
   - 按钮：[取消] [确定删除]

2. **离开确认**：
   - 表单有未保存内容时，离开页面需确认
   - 文案："当前有未保存的内容，确定离开吗？"

## 八、响应式设计

### 断点规则

- **移动端**：< 768px
- **平板**：768px - 1024px
- **桌面端**：> 1024px

### 适配要求

1. **移动端**：
   - 表格改为卡片列表
   - 弹窗改为全屏显示
   - 操作按钮改为底部固定

2. **平板**：
   - 保持桌面端布局
   - 调整间距和字号

## 九、禁止行为清单

### 严格禁止

1. ❌ 表格 CRUD 使用同级表单块
2. ❌ 删除操作无二次确认
3. ❌ 表单提交无加载状态
4. ❌ 操作无反馈提示
5. ❌ 弹窗无关闭按钮
6. ❌ 表单无校验规则
7. ❌ 列表无分页
8. ❌ 空状态无提示

### 不推荐

1. ⚠️ 弹窗嵌套弹窗（最多 2 层）
2. ⚠️ 表单字段过多（> 15 个考虑分步）
3. ⚠️ 操作按钮过多（> 3 个使用下拉）
4. ⚠️ 表格列过多（> 8 列考虑隐藏或分页）
